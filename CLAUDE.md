# CLAUDE.md — FloristApp
## Проект
PWA для управления цветочным магазином. Один пользователь, только телефон.
Документация: `/docs/PRD.md`

## Стек
- React + Tailwind CSS (mobile-first PWA)
- **Слой данных:** localStorage mock (`src/lib/supabase.js`) — временно, до восстановления Supabase (PostgreSQL)
- Vitest + React Testing Library
- Vercel (деплой)

## Язык интерфейса
Русский — все тексты, лейблы, сообщения об ошибках.

## Правила разработки
- Mobile-first всегда. Никакого десктоп-лейаута в v1.
- Tailwind только для стилей — никакого inline CSS.
- Компоненты — функциональные, с хуками.
- Каждый новый компонент = тест в Vitest.
- Запись данных — только при наличии интернета. Офлайн = только просмотр.

## Структура проекта
```
/docs          — PRD и документация
/src
  /components  — UI компоненты
  /pages       — экраны приложения
  /hooks       — кастомные хуки
  /lib         — клиент данных (mock/supabase), утилиты
  /tests       — тесты
```

## Навигация (4 вкладки нижнего таббара)

| Вкладка | Страница | Под-табы |
|---|---|---|
| Остатки | `StockPage` | — |
| Заказы | `OrdersPage` | Заказы / Клиенты |
| Поставки | `DeliveryPage` | Поставки / Поставщики |
| Списание | `WriteOffPage` | — |

- `NewOrderPage` — не таб, открывается из вкладки «Заказы» (кнопка «Создать заказ»)
- `SuppliersPage` — рендерится внутри `DeliveryPage` как под-таб «Поставщики»

## Слой данных: localStorage mock
`src/lib/supabase.js` — полный mock Supabase API (select/insert/update/delete, nested select, view `flower_stock`).
Данные хранятся в `localStorage` под ключом `floristapp_db`.
Сброс: `window.resetFloristDb()` в консоли браузера.
При восстановлении Supabase — заменить файл на оригинальный `createClient(...)`, хуки не трогать.

## Известные мёртвые хуки (не использовать)
- `useDefect` — функция «Брак» убрана из UI (кнопка удалена из `DeliveryCard`). Хук остался, но не подключён.

## Работа с задачами
Реализуй по одному изменению за раз.

### Процесс закрытия задачи (обязательно, в этом порядке)
1. **Тесты** — запустить `npm test`. Все тесты должны быть зелёными.
2. **Ревью кода** — перед коммитом запустить `/simplify` на изменённых файлах. Проверить: нет лишней сложности, нет inline CSS, компоненты функциональные, стили только через Tailwind.
3. **Коммит** — только после прохождения тестов и ревью.
4. **Проверка** — после пуша написать краткий отчёт: что сделано, как проверить. Ждать подтверждения.
