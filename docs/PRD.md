# PRD — FloristApp v1.0  ## Проблема Алина управляет цветочным магазином через бумажный блокнот и Telegram. Нет видимости остатков, резервов, истории движения цветов и сроков свежести партий.  ## Пользователь Один человек (Алина). Устройство — только телефон. Технический уровень — базовый.  ## Цели продукта 1. Заменить блокнот на мобильное PWA 2. Видеть остатки в реальном времени (свободно / зарезервировано) 3. Фиксировать заказы, считать расход цветов автоматически 4. Управлять поставками, браком, поставщиками 5. Отслеживать историю движения по цветку (в какие букеты шёл) и по партии (откуда пришла, куда ушла) 6. Видеть какие партии скоро завянут — пустить в витрину первыми (FIFO) 7. Простой отчёт: что заказывать на следующей неделе  ## Основные сущности  | Сущность | Описание | |---|---| | Цветок | Вид, суммарный остаток, зарезервировано, свободно | | Партия | Поставщик, дата поставки, количество, статус свежести | | Поставщик | Название, контакт (до 5 шт) | | Заказ | Клиент, тип получения, состав букета, статус | | Клиент | Имя, телефон, состав букета, дата готовности, адрес (опционально) | | Брак | Тип (гнилой / не тот цвет), решение (возврат / скидка) | | Движение (лог) | Привязан к цветку И партии |  ## Потоки движения
Закупка: заказ поставщику → предоплата → поставка → приёмка на склад (партия)
Продажа: заказ клиента → резерв цветов → оплата → готов → выдан / доставлен
Списание: брак / увядание → списание из партии
 ## Статусы заказа
Самовывоз: новый → в работе → готов к выдаче → выдан клиенту Доставка: новый → в работе → готов к доставке → доставлен клиенту
Тип получения: `самовывоз` / `доставка` - доставка → адрес обязателен - самовывоз → адрес скрыт  ## Ключевые экраны 1. **Главная** — остатки: свободно / зарезервировано / ⚠️ скоро завянет 2. **Новый заказ** — цветы, количество, клиент, тип получения, дата 3. **Поставка** — принять товар, отметить брак, дата партии 4. **История** — движение по цветку или по партии 5. **Заказы** — список активных 6. **Поставщики** — справочник 7. **Отчёт** — цветы с остатком 0 или < 5 стеблей + активные заказы на 7 дней вперёд  ## Ключевая логика
Свободно = Всего - Зарезервировано Свежесть = текущая дата - дата поставки партии Приоритет витрины = старейшая партия сначала (FIFO) Порог отчёта = 5 стеблей (фиксировано)
Предупреждение если: свободно < 0 | партия > 5 дней.  ## Брак - Гнилой → возврат поставщику, списание из остатка - Не тот цвет → скидка от поставщика, остаётся с пометкой  ## Надёжность - Данные в Supabase — не теряются при смене телефона - Офлайн: просмотр остатков и заказов работает без интернета - Запись (новый заказ, поставка, брак) — только при наличии интернета  ## Стек - Frontend: React + Tailwind (PWA, mobile-first) - Backend: Supabase (PostgreSQL) - Деплой: Vercel  ## Вне скоупа v1 - Финансовая отчётность - Уведомления клиентам - Интеграция с Instagram - Многопользовательский режим
