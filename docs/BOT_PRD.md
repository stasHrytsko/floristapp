# PRD — FloristApp Bot v2.0 (Telegram Integration)
## Контекст
Дополнение к FloristApp v1.0. Та же база данных Supabase.
PWA = витрина и аналитика. Telegram Bot = ввод И вывод данных на ходу.
## Пользователь
Один человек (Алина). Только её Telegram аккаунт имеет доступ к боту (whitelist по Telegram ID).
## Команды бота
### Ввод — Поставки
```
"пришло 30 роз от Марины"
"от Карлоса 20 тюльпанов и 15 пионов"
```
→ создаёт партию с датой сегодня
→ если поставщик не найден точно — бот просит уточнить название как в базе
### Ввод — Брак
```
"5 роз гнилые от Марины"
"от Карлоса розы не тот цвет"
```
→ создаёт запись брака, списывает или помечает
### Ввод — Заказы
```
"заказ Оля +34612345678 самовывоз 12.02.2026 15:00: роза красная 3шт, пион белый 2шт"
"заказ Иван +34698765432 доставка 13.02.2026 11:00 ул.Главная 5: тюльпан 10шт"
```
→ дата и время обязательны
→ состав букета обязателен
→ если цветов не хватает — бот отказывает с предупреждением, заказ не создаётся
### Ввод — Статусы
```
"Оля забрала букет"
"доставили Ивану"
```
→ обновляет статус заказа
### Вывод — Остатки
```
"остатки"
"сколько красных роз?"
"что заканчивается?"
```
→ бот отвечает текстом с актуальными данными из Supabase
### Вывод — Заказы
```
"заказы на сегодня"
"заказы на завтра"
"активные заказы"
```
→ бот отвечает списком заказов со статусами и составом букетов
## Уведомления (автоматические)
- Партия > 5 дней → "⚠️ розы от Марины (10.03) скоро завянут"
- За 24 часа до даты/времени готовности → "📦 завтра в 15:00 букет для Оли: роза красная 3шт, пион белый 2шт"
## Правила
- Поставщик — точное совпадение с именем в базе. Не найден → бот просит уточнить
- Состав букета обязателен при создании заказа
- Дата и время заказа обязательны. Формат: ДД.ММ.ГГГГ ЧЧ:ММ
- Нехватка цветов → заказ не создаётся, бот пишет предупреждение
- Новые поставщики — только через PWA
## Архитектура
```
Telegram → Webhook → Node.js сервис → Supabase
PWA → Supabase (та же БД)
Уведомления: Supabase Edge Function → Telegram Bot API
```
## Стек
- Node.js + Telegraf (Telegram bot framework)
- Supabase (та же БД что в v1)
- Деплой: Railway
## Вне скоупа v2
- Голосовые сообщения
- Фото букетов через бот
- AI-интерпретация свободного текста (v3)
- Несколько пользователей
