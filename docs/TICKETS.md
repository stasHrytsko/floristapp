# FloristApp — Тикеты

## #1 — Инициализировать React + Tailwind проект

Создать React проект с Tailwind CSS. Настроить PWA манифест (manifest.json, service worker).

**Структура папок:**
- /components
- /pages
- /hooks
- /lib
- /tests

---

## #2 — Подключить Supabase, создать схему БД

Создать проект в Supabase.

**Таблицы:** flowers, batches, suppliers, orders, order_items, defects, movements

Настроить supabase клиент в `/lib/supabase.js`

---

## #3 — Настроить PWA конфигурацию

- manifest.json: название FloristApp, иконки, theme_color
- Service worker: кэширование для офлайн-просмотра
- Офлайн: только чтение. Запись — только при наличии интернета.

---

## #4 — Настроить Vitest

Установить Vitest + React Testing Library. Базовый тест-конфиг. Smoke test на App компонент.

---

## #5 — Экран: Главная (остатки)

- Список цветов: название / свободно / зарезервировано
- Предупреждение ⚠️ если партия > 5 дней (FIFO)
- Предупреждение если свободно < 0
- Данные из Supabase realtime

---

## #6 — Экран: Поставщики

- Список поставщиков (макс 5): имя + телефон
- CRUD: добавить / редактировать / удалить

---

## #7 — Экран: Поставка

- Выбрать поставщика, дату, список цветов + количество
- Отметить брак: гнилой (списание) / не тот цвет (скидка, остаётся)
- Создаёт партию с датой поставки
- Лог движения: поставка → склад

---

## #8 — Экран: Новый заказ

- Поля: имя клиента, телефон, дата готовности
- Тип получения: самовывоз / доставка
- Адрес: показывать только если доставка
- Состав букета: выбрать цветок + количество (только из свободных)
- Лог движения: заказ → резерв

---

## #9 — Экран: Заказы

- Список активных заказов с статусом
- Самовывоз: новый → в работе → готов к выдаче → выдан
- Доставка: новый → в работе → готов к доставке → доставлен
- Смена статуса одним тапом

---

## #10 — Экран: История

- Фильтр: по цветку ИЛИ по партии
- По цветку: в какие букеты шёл
- По партии: откуда пришла, куда ушла, брак
- Хронологический лог движений

---

## #11 — Экран: Отчёт

- Цветы с остатком 0 или < 5 стеблей
- Активные заказы на ближайшие 7 дней
- Вывод: что нужно заказать и сколько

---

## #12 — Vercel deploy + тест на телефоне

- Подключить репо к Vercel
- Deploy на prod
- Открыть на телефоне Алины, проверить PWA установку
